Introduction

 [toc]
This gives BeOS users the world over a very nice GUI for encoding Ogg Vorbis files. It requires oggenc 0.7 (available at BeBits). The file can be on any drive that you can run a query on (basicly this means you can have it anywhere on any mounted bfs volume), but we recommend installing in /boot/home/config/bin/ (a link to which has also been included in the base FlipSide A.E. distribution). oggenc was compiled from the beta4 release of the Ogg Vorbis libraries. Just a warning: this encoder is not fast. From my rudamentary benchmarking, it will probably take 4x longer to encode it with oggenc than with gogo. This is a mixture of the tools being very young, and gcc for BeOS not being completely up to date. But if you difintely want to get away from the money hungry people that "own" the mp3 spec, ogg is definitely the way to go. And the Ogg Encoder add-on for FlipSide A.E. is the best way to get there!




Encoder Settings

 [toc]
Here are a brief explination of the settings available to you. This covers all of the good stuff that oggenc would allow you to do on the command line.
  • Bitrate - Allows you to choose a mean bitrate. All Ogg Vorbis files are encoded at a variable bitrate. The rate specified on the menu only reflects an average.

Details Editor

 [toc]
Ogg Encoder writes both BeOS attributes, and vorbis comments to the newly encoded file. Vorbis comments use a text based solution for genre comments, so dream up your own genres all day long! Unfortunately, The Army Knife does not support Vorbis comments at this time. It will append an ID3 tag to the end of the file if you choose to write tags. This should be remedied soon.
