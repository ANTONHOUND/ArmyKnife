Introduction

 [toc]
This add-on, along with FlipSide A.E. finally gives BeOS users the ability to encode MP3's with lame from a friendly GUI. This release of MP3 Lame has been tested with lame version 3.88 MMX (beta 1), if you are using an older version, it is recomended that you upgrade. The binary is available on BeBits.




Encoder Settings

 [toc]
Here are a brief explination of the settings available to you. This covers all of the good stuff for encoding that lame would allow you to do on the command line.
  • Bitrate - Allows you to choose from either a fixed bitrate, or use one of the levels of varible bitrate compression. This uses lame's new VBR code (--vbr-new).
  • Output Format - Allows you to specify the output format of the file (stereo,mono,j-stereo).
  • Psycho Acoustics - Allows you to specify the Psycho Acoustic algorithm to be used. This is the -q switch for lame. 2 is equal to -h and 7 is equal to -f.




File Path and Name Pattern

 [toc]

Details Editor

 [toc]
MP3 Lame writes both BeOS attributes, and ID3 v1.1 tags to the newly encoded file. Remember, if you specified a genre other than an ID3 v1.1 genre, the ID3 tag will reflect the "Other" genre, but the BeOS attribute will be what you specified. Very compatible with The Army Knife if you wish to update things later!
